Updated Guest Journey
Phase 1 – Invitation

Guest opens their personalized invitation.

Opens the envelope
Views invitation
Clicks RSVP
Selects number of attendees
Enters attendee names
Receives:
RSVP confirmation
Serial number
QR code (or access code)

At this point, no seat number is assigned yet.

Instead, show a notice like:

Your seat number will be available one week before the wedding. Please revisit this invitation using this same link to view your assigned table and seat.

Phase 2 – One Week Before the Wedding

You finalize seating.

The system updates each guest's invitation automatically.

When they revisit their personalized link, a prominent banner appears at the bottom of the screen.

For example:

══════════════════════════════
      TABLE F3
      SEATS 5–7
══════════════════════════════

Or, if it's an individual seat:

══════════════════════════════
       TABLE V2
        SEAT 4
══════════════════════════════

I would make this a sticky footer so it's always visible as they scroll.

You could also use a subtle animation (fade or pulse) the first time it appears so guests notice it without being distracting.

Phase 3 – Email Notification (Optional)

For guests who provided an email address:

Your seat has been assigned. Click below to view your invitation and seating details.

This doesn't cost anything beyond using a standard email service.

I would make one small improvement

Instead of assigning seat numbers, consider assigning table numbers for most guests.

Example:

Table 12

rather than

Seat 5

Unless your reception has very formal seating, most Nigerian weddings don't require exact seat positions. Ushers can guide guests to available chairs at their assigned table.

Benefits:

Easier to manage.
Easier to handle late arrivals.
Easier if someone swaps chairs.
Less rework if a guest cancels.

For VIPs or the bridal party, you can still assign exact seats if needed.

Another feature I'd add

One more suggestion that could save you a lot of stress

Build an Admin Seating Planner into the dashboard.

Instead of editing seat numbers directly in the database, create a visual planner where you can:

Create tables (VIP 1, Family A, Friends B, etc.).
Specify each table's capacity (8, 10, or 12 seats).
Drag and drop families or guests onto tables.
Keep family members together automatically.
See live occupancy (e.g., Table F3: 8/10 seats filled).

Once you're satisfied, click "Publish Seating".

Only then do seat assignments become visible on each guest's invitation.

This gives you complete freedom to make changes until you're ready, and guests always see the latest information through their personalized invitation link. I think it's the cleanest and most practical solution for your wedding website.


Design Style

Use a modern SaaS aesthetic:

Clean white background
Light gray borders (#E5E7EB)
Soft shadows
Rounded corners (10–12px)
wine as the primary accent color (matching wedding website theme)
Consistent spacing (8px spacing system)
Clear typography with good hierarchy
Hover states for interactive elements
Smooth transitions (150–200ms)
Accessible color contrast and keyboard navigation

The interface should feel closer to Linear, Stripe Dashboard, Notion, or Vercel than to older enterprise. Prioritize speed, clarity, and ease of use over feature density.
responsive both on mobile and desktop.
